# Code of Conduct

## Standards

Project discussion should stay technical, respectful, and focused on improving
the software.

Examples of expected behavior:

- Give actionable feedback.
- Assume mistakes are fixable.
- Respect privacy and do not request secrets, private keys, or subscription URLs.
- Keep security reports factual and reproducible.

Examples of unacceptable behavior:

- Harassment or personal attacks.
- Publishing private credentials or user data.
- Intentionally bypassing security checks.
- Disruptive or bad-faith participation.

## Enforcement

Maintainers may remove comments, reject contributions, or block participants who
violate this code of conduct.

## Reporting

Report issues privately to the project owner. Include enough technical detail to
reproduce security or safety concerns without exposing secrets.
