# Internationalization

Translations are handled by `internal/i18n` and the static UI translation layer.

## Adding A String

1. Add a stable key to the translation catalog.
2. Add values for every supported locale.
3. Use `tr("key")` in JavaScript or the Go i18n helper in backend-facing text.
4. Add a test when the string is part of onboarding, diagnostics, tray, or
   notification behavior.

## JavaScript Rules

- Do not hardcode new user-visible Russian or English strings in UI modules.
- Use `esc()` before inserting translated or user-controlled text into HTML.
- Use locale-aware date formatting helpers instead of raw `toLocaleString`.

## Backend Rules

Backend errors that are directly shown to the user should use localized message
helpers or stable error codes that the UI can translate.

## Testing

Run:

```powershell
go test ./internal/i18n ./internal/api -count=1 -race -timeout=180s
```

The i18n tests guard against accidental untranslated Russian literals in
localized JavaScript files.
