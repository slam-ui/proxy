### 5.3 Общий Handler middleware

Текущая проблема: Каждый handler проверяет auth отдельно

```go
func HandleServersGet(w http.ResponseWriter, r *http.Request) {
    if !isAuthed(r) { handleUnauth(w); return }  // repeat in every handler
    // ...
}
```

**Решение:**
```go
// internal/api/middleware.go
func AuthMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        if !isAuthed(r) { http.Error(w, "Unauthorized", 401); return }
        next.ServeHTTP(w, r)
    })
}

// Usage:
mux.Handle("/api/servers", AuthMiddleware(http.HandlerFunc(HandleServersGet)))
```

**Преимущества:**
- ✅ DRY (Don't Repeat Yourself)
- ✅ Centralized auth logic
- ✅ Меньше ошибок

## 📚 Справочные команды

### Проверить test покрытие

```powershell
.\run_tests.ps1 coverage
# Открыть coverage.html в браузере
```

### Посчитать строки кода

```powershell
Get-ChildItem -Recurse -Include "*.go" | 
  Where-Object { $_.Name -notmatch "_test" } | 
  Measure-Object -Line | 
  Select-Object @{N='Lines';E={$_.Lines}}
```

### Найти дублирующиеся функции в тестах

```bash
grep -r "^func Test" internal/proxy/manager*_test.go
```

### Запустить конкретные тесты

```powershell
go test -v -run "TestManager" ./internal/proxy

# С race detector
go test -race -v -run "TestManager" ./internal/proxy -timeout 30s
```
